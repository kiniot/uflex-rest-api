# Error Codes

This document lists the API error `code` values currently used by the backend, grouped by bounded context.

The JSON error shape is:

```json
{
  "code": "USER_WITH_ID_NOT_FOUND",
  "message": "User with ID ... not found",
  "status": 404,
  "title": "Not Found",
  "timestamp": "2026-06-14T15:02:59.634156-05:00",
  "path": "/api/v1/users/me"
}
```

## Notes

- Domain exception codes are derived from the exception class name.
- The suffix `Exception` is removed.
- The remaining name is converted to `UPPER_SNAKE_CASE`.
- Some framework or security errors use fixed codes such as `AUTH_REQUIRED` and `ACCESS_DENIED`.

## `HTTP_<STATUS>` Fallback

`HTTP_<STATUS>` is a generic fallback used for `ErrorResponseException` cases whose HTTP status does not have an explicit mapping yet.

Examples:

- `HTTP_422`
- `HTTP_429`
- `HTTP_502`

Current explicit mappings are:

- `400` -> `BAD_REQUEST`
- `401` -> `AUTH_REQUIRED`
- `403` -> `ACCESS_DENIED`
- `404` -> `NOT_FOUND`
- `409` -> `CONFLICT`
